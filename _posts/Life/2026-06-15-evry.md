# Biometrics
1. Pattern Recognition System
   
Real world --> Sensor --> Preprocessing --> Feature Extraction --> Classification ( Model-based | Distance-based)

*Classes: Specific to each Pattern Recognition problem*
*Feature extraction: Handcrafted or Automatic*

2. Template Extraction Step
   
Acquired Signal --> Feature Extraction --> Feature Vector: <<Template>> = a compressed representation of the pattern

- Case of a digitizing tablet:
  - raw data captured:
    - Coordinates (x(t), y(t))
    - Pressure: p(t)
    - Pen-inclination angles
- Case of a Smartphone:
  - Raw data captured:
    - Only (x(t), y(t))

3. Handwritten Signatures

Biometric modality widely used in civilian applications
-> Accepted by the general public
  - Document content approval
  - Financial transactions: checks, credit card transactions
Behavior Biometrics
  - Strong intraclass variability ( one person each time sign a little bit different )
  - Strong time variability (Speed of writing)
  - A person can change his/her signature
  - Several types of forgeries

4. What is Identity Verification
   - Classification problem of 2 classes
   - Verifying if the claimed identity is true or false
2 Types of approaches
   - Model-based: Hidden Markov Models (HMMs), Gaussian Mixture Models (GMMs)
       - Absorb the intra-personal variability (intraclass = the writer): the signatures of a person are variable
       - The model's output: *similarity score* (Likelihood of the test signature given the model)
    - Distance-based: elastic distance (DTW)
        - Can compare sequences of different length (<<elastic>>) -> output: *distance score* (dissimilarity)
        - Tolerates (Stable) with local variability in signatures
5. Feature Extraction Step
   - Local means extracting features pointwise: in most cases (sequence modelling)
   
   - Global means extracting features on the whole signature
       - time-related features as total signature time, ratio of pen-down time to total time, ....
       - dynamoc features as average speed, average horizontal speed, etc, ..
       - shape-related features as length to width ratio, histogram of pathtangent angles.
6. Model-based Verification
   
   (Enrollment) : Data Acquisition (one person) -> Feature Extraction -> Model Training -> Client Model
   
   (Verification) : Data Acquisition (one person) -> Feature Extraction -> Likelihood Computation -> Comparision to threshold -> Decision: Acceptance if Log-lik > threshold | Otherwise: Rejection

  How do we measure performance ?
- 2 types of errors: FRR (False Rejection Rate) and FAR (False Acceptance Rate)
  - FR = False Rejection: falsely rejecting a client
  - FA = False Acceptance: falsely accepting an impostor

    FRR = (Nb of FR)/(Nb of client samples)              ; FAR = (Nb of FA)/(Nb impostor samples)
