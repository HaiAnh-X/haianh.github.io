3. Handwritten Signatures

Biometric modality widely used in civilian applications
-> Accepted by the general public
  - Document content approval
  - Financial transactions: checks, credit card transactions
Behavior Biometrics
  - Strong intraclass variability ( one person each time sign a little bit different )
  - Strong time variability (Speed of writing)
  - A person can change his/her signature
  - Several types of forgeries

4. What is Identity Verification
   - Classification problem of 2 classes
   - Verifying if the claimed identity is true or false
2 Types of approaches
   - Model-based: Hidden Markov Models (HMMs), Gaussian Mixture Models (GMMs)
       - Absorb the intra-personal variability (intraclass = the writer): the signatures of a person are variable
       - The model's output: *similarity score* (Likelihood of the test signature given the model)
    - Distance-based: elastic distance (DTW)
        - Can compare sequences of different length (<<elastic>>) -> output: *distance score* (dissimilarity)
        - Tolerates (Stable) with local variability in signatures
5. Feature Extraction Step
   - Local means extracting features pointwise: in most cases (sequence modelling)
   
   - Global means extracting features on the whole signature
       - time-related features as total signature time, ratio of pen-down time to total time, ....
       - dynamoc features as average speed, average horizontal speed, etc, ..
       - shape-related features as length to width ratio, histogram of pathtangent angles.

7. Distance-based: elastic distance (DTW)
   
**Gaussian Mixture Model**:
$$P(x/\lambda) = \sum_{i=1}^{K}C_iN(x/\mu_i\sum_i)$$
   - $\lambda$ : a mixture of Gaussian densitites
   - $x$: a feature vector
   - $C_k$: is the weight of Mixture Component i
   -> belong to the [0,1] interval
   -> sum up to 1

   - Original motivation for speech:
        - Each component: an acoustic class
        - Acoustic clasees (Vowels, nasals, etc): reflect speaker-dependent vocal tract configurations
        - Client space: a mixture of acoustic classes
   
   GMM: No temporal relation is learned between points

**HMM**: a succession of "stationary parts" <-> each can be modeled by the same probability density

   - HMM -> States -> Stationary parts of the signal
   - HMM -> Transitions -> Variations in time

   - Left-to-Right HMM
      - To model a sequence: one state j is reached, one cannot return to state (j-1)
   - Ergodic HMM
      - To model a sequence: All transitions are allowed

   - Components of a HMM
      - N: number of states in the model -> S ={$$S_1, S_2, ......., S_N$$}
      - A: Matrix of probability transitions -> $$a_{ij} = P[q_{t+1} = S_j|q_t=S_)i], 1 \leq i, j \leq N$$
      - Initial distribution of the states -> $$\pi_i = P[q_1 = S_i], 1 \leq i \leq N$$
      - Emission law in each state -> $$B_j(O_t) = P[O_t|q_t = S_j] , 1 \leq j \leq N$$
           - Discrete HMM: $B_j$ is a matrix. 1 element of such matrix = the emission probability of a symbol ( column) by a state (row)
           - Continuous HMM: $B_j$ is a mixture of gaussian probability density functions
             
      $\rightarrow$ the HMM writter model $\lambda = (A, B, \pi)$
   - Modeling a signature by an HMM:
        - The signature ( a non stationary signal) considered as a piecewise-stationaty signal
        - Modeled by a mixtrue of 3 gaussian densities
        - K-Means with K = 3 for model initialization:
             - 3 mean vectors
             - 3 covariance matrices
